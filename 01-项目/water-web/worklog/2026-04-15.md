## 详细变更说明

### 1. 水质检测记录页整体落地

昨天的核心工作，是把 `src/views/business/water-quality-detection/` 这一整页从零补起来。

页面层提供了这些基础能力：

- 顶部按年份筛选，默认带当前年份
- 主表展示港区、泊位、采样点、检测日期、检测项、周期、完成结果、备注、操作时间、操作人
- 行操作支持修改、删除、完成、撤销、附件管理
- 工具栏提供“批量生成”和“新增”两个主入口

当前实现延续项目既有模式：`index.vue` 只负责模板，业务逻辑全部下沉到 `utils/hooks.tsx`。

### 2. CRUD 与状态流转补齐

`utils/api.ts` 和 `utils/hooks.tsx` 这轮把 `BDet` 相关的主流程都补齐了：

- 列表查询走 `/api/BDet`，页面层当前只按 `Year` 查整年数据
- 新增、修改共用一套请求体拼装逻辑，修改时额外补 `id`
- 删除接口按后端要求走 query 参数 `id + username`
- 完成/撤销走 `/api/BDet/FinishAndRevoke`，只切 `finish_mark`，展示文案仍以接口返回为准

这意味着这一页已经不只是静态表格，而是把最基本的业务闭环先跑通了。

### 3. 表单层把接口格式差异收口

`form.vue` 主要做了两类收口：

- 检测日期界面上使用月份选择器，提交时再统一补成 `YYYY-MM-01`
- 港区下拉直接提交中文名称，不走常规的 code/name 分离模式，保持和接口口径一致

另外，`berthno`、`sampno`、`samp_item`、`remark` 在提交前统一做了 trim，避免因为录入空格把脏值送到后端。

### 4. 批量生成按“模板年份 -> 目标年份”处理

`batch-form.vue` 和 `openBatchDialog` 这一套逻辑，主要是把批量生成入口补齐：

- 表单只保留两个字段：`oldyear` 和 `newyear`
- 弹窗打开时默认把 `newyear` 带成当前列表筛选年份
- 提交后调用 `/api/BDet/CreateBatch`
- 成功后把页面筛选年份同步切到目标年份并刷新列表

这样做的好处是操作路径够短，用户不用在弹窗和筛选器之间来回对齐年份。

### 5. 附件管理弹窗先做成可用版本

`accessory.vue` 昨天补了完整的附件管理基础能力，当前约定一个记录有 `annex1`、`annex2`、`annex3` 三个附件位：

- 表格按 3 个固定附件位展示当前文件名、后缀和操作按钮
- 上传时按接口要求拼 `detpid`、`sampdate(yyyyMMdd)`、`sampname` 和 `type`
- 下载接口传的是服务端存储键，浏览器落盘时再还原成“展示名 + 后缀”
- 删除时除记录 id 外，还要把附件槽位和当前存储键一并传给后端
- 上传/删除成功后，会再按记录 id 回查单条数据，避免父表格里的附件字段不是最新值

这部分已经能支撑基础联调，后面如果要扩展预览能力，再往上叠即可。

## 遇到的问题

### 问题1：表单展示值和接口提交值不完全一致

- **现象**：检测日期界面只需要选到月份，但接口显然不是只收 `YYYY-MM`
- **解决方案**：在 `form.vue` 的 `getFormData()` 里统一补成 `YYYY-MM-01`，不把格式转换逻辑散落到模板层

### 问题2：港区字段不能按常规字典 code 提交

- **现象**：按常规做法提交字典 value 容易和后端真实需要的中文港区名不一致
- **解决方案**：港区选项的 `label` 和 `value` 都直接映射成 `zonE_NAME`，保证选中即提交中文

### 问题3：附件接口参数和主表记录字段口径不一致

- **现象**：附件上传、下载、删除不只是传记录 id，还依赖检测日期、采样点和附件槽位/存储键
- **解决方案**：在附件弹窗内部统一封装参数拼装和回查逻辑，避免父层到处关心附件接口细节

## 技术决策

- **决策**：继续沿用“`index.vue` 只放模板，`utils/hooks.tsx` 承担业务逻辑”的页面组织方式
- **原因**：和仓库现有业务页一致，后续维护成本最低

- **决策**：附件位先按固定三槽实现，而不是动态附件列表
- **原因**：后端字段就是 `annex1~3`，前端先贴合接口落地，避免过度抽象

- **决策**：表单层统一吸收日期、港区、trim 等格式差异
- **原因**：页面逻辑可以保持简单，接口适配点也更集中
