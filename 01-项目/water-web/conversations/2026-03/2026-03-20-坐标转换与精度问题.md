# 坐标转换与精度问题 - 对话记录

**标签**：#conversation #water-web

## 问题/需求

连岛的墨卡托坐标 (13293816.71, 4125781.58) 被误判为超出中国范围，需要排查原因并解决。

同时 TileInfo 配置中的 scale 值导致 TypeScript 精度丢失警告。

---

## 关键回答

### 要点一：坐标系混淆

坐标值 13293816.71 不是经纬度，而是 **Web 墨卡托投影坐标**（EPSG:3857）。转换后经度约 119.5°E，纬度约 34.7°N，对应江苏省连云港市连岛，完全在中国范围内。

### 要点二：墨卡托转经纬度公式

```javascript
const MAX_EXTENT = 20037508.34;

function mercatorToLatLng(x, y) {
  const lng = (x / MAX_EXTENT) * 180;
  const lat =
    (2 * Math.atan(Math.exp((y / MAX_EXTENT) * Math.PI)) - Math.PI / 2) *
    (180 / Math.PI);
  return { lng, lat };
}
```

### 要点三：批量转换二维数组

```javascript
const latLngArray = coordinates.map(([x, y]) => mercatorToLatLng(x, y));
```

### 要点四：JS 精度限制

JS 浮点数精度约 16 位有效数字，TileInfo 中的 scale 值（30位）超出限制，需截断。

---

## 决策要点

- **坐标转换方案**：使用纯数学公式而非 proj4js 库，减少依赖
- **精度处理**：截断 scale 到 16 位，对瓦片地图显示无影响
