## 遇到的问题

### 问题1：首页接口返回形态并不完全统一

- **现象**：部分接口是标准业务包装结构，部分文档示例更接近裸对象/数组
- **解决方案**：在 `api.ts` 中做兼容性类型定义，避免组件层把“有数据”误判成“空数据”

### 问题2：页面模块过多，继续堆在 `index.vue` 会迅速失控

- **现象**：首页同时承载地图、详情轮播、五个统计面板和顶部状态栏
- **解决方案**：把统计区拆成独立组件，`index.vue` 负责布局编排，面板内部各管各的数据和展示

---

## 技术决策

- **决策**：首页统计模块拆成独立组件
- **原因**：降低 `index.vue` 复杂度，便于后续逐块联调和替换

- **决策**：为首页接口单独补充类型定义，而不是直接复用 `any`
- **原因**：接口字段多且口径不统一，先在 API 层收口更稳

- **决策**：增加 Vue DevTools 开关，而不是写死启用逻辑
- **原因**：开发调试需要便利，但构建配置要可控，不能一把梭
