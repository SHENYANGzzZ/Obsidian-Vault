## 详细变更说明

### 1. GIS区域颜色样式支持

在区域管理功能中新增颜色样式调整能力：

- `areaForm.vue`：新增颜色选择器，支持区域填充色和边框色自定义
- `areaAddForm.tsx`：同步新增颜色参数字段
- `index.vue` + `maphook.ts`：地图渲染时应用自定义颜色，修复了区域参数传递异常导致颜色不生效的 bug

### 2. 全局分页配置统一

涉及 14 个业务模块的 hooks.tsx 文件，统一调整分页组件的：

- **pageSize 默认值**：统一为合理的默认分页数
- **分页组件页数选项**：保持全局一致性

涉及模块包括：分析节点设置、公司分组、连接线管理、设备管理、管线资料、节点资料、港区管理、抄表周期、抄表计量、操作日志、费率系数等。

### 3. 打包优化 — 自动去除调试代码

在 `vite.config.ts` 中新增 terser 压缩配置：

```typescript
// 生产环境自动去除 console 和 debugger
build: {
  minify: 'terser',
  terserOptions: {
    compress: {
      drop_console: true,
      drop_debugger: true
    }
  }
}
```

**效果**：生产包中不再包含调试代码，提升安全性和性能。

### 4. TypeScript 代码修复

修复 `fix(): 修复无法到达的代码` — 清理了条件判断后的死代码分支，消除 TypeScript 编译警告。

## 遇到的问题

### 问题1：GIS区域颜色参数传递异常

- **现象**：设置区域颜色后，地图渲染不生效
- **原因**：`maphook.ts` 中区域参数拼接时，颜色字段未正确传入渲染函数
- **解决方案**：重构区域参数传递链路，确保颜色值在整个渲染流程中正确传递

---

## 技术决策

- **决策**：使用 terser 替代 esbuild 做生产环境压缩
- **原因**：terser 支持更精细的 `drop_console`/`drop_debugger` 配置，esbuild 的 pure 注释方式不够直观
- **影响**：生产构建时间略有增加，但代码安全性提升

### 额外问题记录

#### 问题2：坐标超出中国范围误判

- **原因**：代码将墨卡托投影坐标 (EPSG:3857) 当作经纬度 (EPSG:4326) 处理
- **解决方案**：添加坐标转换函数 `mercatorToLatLng`，在判断前先逆转换

#### 问题3：TileInfo scale 精度丢失

- **原因**：scale 值小数位过多，超出 JavaScript 浮点有效精度
- **解决方案**：将 scale 截断到 16 位有效数字

### 补充技术决策

- **决策**：使用 `mercatorToLatLng` 进行坐标逆转换
- **原因**：无需引入额外依赖，适合当前 Web 墨卡托场景
