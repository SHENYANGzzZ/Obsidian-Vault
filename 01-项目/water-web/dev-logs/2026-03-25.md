## 接口对接（按文档规范）

### 1. API 接口路径

| 接口名称     | 方法   | 路径                          |
| ------------ | ------ | ----------------------------- |
| 公司节点查询 | GET    | /api/GDotComp                 |
| 公司节点增加 | POST   | /api/GDotComp                 |
| 公司节点修改 | PUT    | /api/GDotComp                 |
| 公司节点删除 | DELETE | /api/GDotComp/{id}            |
| 附件上传     | POST   | /api/WRepair/uploadfilecomp   |
| 附件下载     | GET    | /api/WRepair/downloadfilecomp |
| 附件删除     | DELETE | /api/WRepair/deletefilecomp   |

### 2. 查询参数

| 参数      | 位置  | 说明                 |
| --------- | ----- | -------------------- |
| id        | query | 节点ID，为空时取全部 |
| pageSize  | query | 分页大小，默认50     |
| pageIndex | query | 页码                 |

### 3. 新增/修改参数

**Query**: compid（公司ID，必填）

**Body**:
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| doT_CODE | string | 是 | 节点代码 |
| doT_NAME | string | 是 | 节点名称 |
| latitude | number | 否 | 显示纬度 |
| longitude | number | 否 | 显示经度 |
| latitudE_F | number | 否 | 实测纬度 |
| longitudE_F | number | 否 | 实测经度 |
| remark | string | 否 | 备注 |
| useR_NAME | string | 否 | 操作人 |

### 4. 附件管理参数

**上传**:
| 参数 | 位置 | 说明 |
|------|------|------|
| compid | query | 公司ID |
| compname | query | 公司简称 |
| type | query | 照片类型：1=PIC1, 2=PIC2, 3=PIC3 |

**下载**: fileName（PIC1/PIC2/PIC3储存文件名）

**删除**: compid + type + fileName

> 时间：2026-03-25 16:30+

### 问题排查：附件管理上传按钮无响应

**现象**：公司节点资料中的附件管理弹窗，点击上传后没有触发任何接口调用。

**排查过程**：

1. 检查 `accessory.vue` 组件，发现上传事件绑定存在问题
2. 进一步分析：**公司节点资料新增时，公司不是必填项**，导致节点可能没有公司信息（`compid` 为空）
3. 后端接口明确限制：上传附件时必须绑定公司信息，无公司 ID 的节点不允许上传

**业务规则确认**：

- 新增节点时公司字段非必填（节点可以独立存在）
- 附件上传强依赖公司信息（`compid` + `compname`）
- 因此：**无公司信息的节点不能上传照片**，前端应相应禁用上传按钮并给出提示

**遗留问题**：

- 有公司信息的节点也无法点击上传按钮 → 该问题留到次日（2026-03-26）继续排查
