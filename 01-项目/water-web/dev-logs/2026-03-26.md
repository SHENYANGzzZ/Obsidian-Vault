## 遇到的问题

### 问题1：删除接口缺少错误处理

- **原因**：旧代码的 `.catch` 直接解构 `response.data`，接口异常时会报错
- **解决方案**：改为安全访问 `error?.response?.data?.message`，并提供默认错误提示

### 问题2：compid 传参方式不对

- **原因**：文档明确 compid 是 Query 参数而非 Body 参数
- **解决方案**：新增/修改接口将 compid 从 Body 移到 Query 参数

## 补充：上午会话调试记录

> 时间：2026-03-26 08:43+

### 问题排查：附件上传一直失败（接续 2026-03-25 遗留问题）

**背景**：昨天发现有公司信息的节点也无法正常上传照片，今早继续排查。

**任务**：检查 `src/views/business/company-node/accessory.vue` 组件上传文件失败问题，同时将上传文件类型限制为图片类型。

**根因定位（关键发现）**：

> 封装的请求函数（Axios 封装层）中 `Content-Type` 默认设置为 `application/json`，导致 `FormData` 文件上传请求被强制序列化为 JSON 格式，服务端无法解析上传内容。

**解决方案**：

- 文件上传接口调用时，手动指定 `Content-Type: multipart/form-data`，或让浏览器自动设置（不传 Content-Type，由 FormData 自动携带 boundary）
- 在 `accessory.vue` 中调用上传 API 时，通过请求配置覆盖默认 Content-Type

**同步完成**：将上传文件类型限制为图片类型（`accept="image/*"`），防止用户误传非图片文件。

### 技术总结

| 问题                 | 原因                                                  | 解决                              |
| -------------------- | ----------------------------------------------------- | --------------------------------- |
| 文件上传返回接口错误 | Axios 拦截器全局强制 `Content-Type: application/json` | 上传接口单独配置覆盖 Content-Type |
| 上传文件类型无限制   | 未设置 `accept` 属性                                  | 添加 `accept="image/*"` 限制      |
