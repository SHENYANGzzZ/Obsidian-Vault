## 🎯 目标

在现有GIS地图基础上，实现以下三个核心功能增强：

1. 管道粗细区分管径大小
2. 管道流向有流动的效果
3. 直接在地图上开关阀门

---

## 🔍 现状分析

### 当前管道样式逻辑

- **粗细区分**：主地图仍基于`m_LEVEL`字段（0:主管, 1:支管, 2:小支管）
- **颜色区分**：基于`m_LEVEL`字段（主管蓝色，支管绿色，小支管红色）
- **流向显示**：主地图保留箭头指示，且已增加基于 `FlowLineLayer` 的粒子流动动画

### 数据可用性

- ✅ **管径字段**：`c_SIZE`（字符串类型，如"DN100"）
- ✅ **流向字段**：`direct`（0:未检测, 1:单向, 2:双向）
- ✅ **阀门控制接口**：`PUT /api/Adjust/OperCtrlDot`（参数：dotid, onoff, username）

### 技术栈

- **地图引擎**：ArcGIS API for JavaScript 4.25
- **主符号系统**：CIMSymbol（主管/支管/末端管线基础样式）
- **流动动画实现**：自定义 Canvas 粒子层 `FlowLineLayer`
- **数据来源**：`getDotDirect()`获取管线连接关系，`getNodeAll()`获取节点数据

## 📈 当前进度同步

截至 **2026-04-08**，本轮 GIS 功能增强的实际进度如下：

- [x] **管道粗细按管径显示**：已完成，主地图已改为按真实 `c_SIZE` 动态映射线宽，并统一静态管线与流动动画的尺寸口径
- [x] **管道流向动画效果**：已完成，采用 `FlowLineLayer` Canvas 粒子动画方案，并已接入页面开关
- [ ] **地图上阀门开关控制**：未完成，`dotSwitch` 接口存在，但尚未接入 ArcGIS 节点弹窗

### 管道粗细按管径显示的实施指导意见

- 将代码里 `originalLineSymbol`、`zhuCS`、`zhiCS`、`xzCS` 这组固定分级样式，改造成按管径动态生成的样式
- 以 `DN100` 到 `DN800` 为主要映射区间，随管径逐级增加管线宽度
- 箭头大小、箭头间距、箭头描边宽度等视觉参数，也要随管径同步放大
- 目标不是只改线宽，而是让线体、箭头、方向表达形成统一的按管径缩放效果
- 落地时优先改 `src/views/business/arcgis/index.vue` 中 `getAllLine()` 对 `originalLineSymbol`、`zhuCS`、`zhiCS`、`xzCS` 的使用方式，避免继续维持固定档位样式
