# 2026-05-09 工作日志

## [[CtnmarketWeb]]

### ctngencargo 客户查询路由拆分与权限收口

- 将 `src/views/ctngencargo` 的客户查询从“单路由 + `query.page` 内嵌组件”调整为“独立路由跳转到对应组件”。
- 件杂货改为独立路由 `/ctngencargo/bb_cust_edit`，集装箱改为独立路由 `/ctngencargo/baseinfo`、`/ctngencargo/baseinfoedit`。
- 将旧的内嵌新增/编辑组件引用从 `cust_query.vue` 中移除，并删除已不再使用的旧文件。
- 统一整理复制后的组件命名，集装箱组件改为下划线格式：`base_info.vue`、`base_info_edit.vue`。
- 补齐返回列表页逻辑，回到 `/ctngencargo/cust_query` 时可读取浏览器缓存并恢复 `BUSI_TYPE`。

### ctngencargo 权限规则改造

- 将按钮权限改造为统一规则：
- 查询：`bb-manager`、`bb-area-manager`、`bb-grid-manager`、`bb-grid-officer` 四个角色都可见。
- 新增 / 修改 / 删除：仅 `bb-area-manager`、`bb-grid-manager`、`bb-grid-officer` 可用。
- 审核能力预留为 `bb-area-manager`、`bb-grid-manager` 两个角色。
- 删除旧的 `canEdit` 与 `api/mpa/sys_userinfo` 二次请求判断，改为直接基于 `funcs` 计算 `canQuery / canMutate / canReview`。

### ctngencargo 接口替换梳理

- 根据后端截图整理了 `BbCustInfoController`、`CmmCustAcvController`、`CmmCustArchivesController` 的新接口清单。
- 盘点了当前 `src/views/ctngencargo` 下实际涉及的接口，区分出“可直接替换”和“暂不能直接替换”的部分。
- 可直接替换的重点包括：件杂货详情 / 办事处接口、集装箱新增 / 详情 / 修改接口。
- 暂不直接替换的重点包括：统一分页接口 `CUSTOMER/PAGE`、集装箱删除接口，以及业务配置 / 母公司搜索等暂未看到新接口映射的部分。
- 已将这部分整理为单独计划文档：[[01-项目/CtnmarketWeb/plans/2026-05-09-ctngencargo-api-migration-plan|ctngencargo 路由迁移与接口替换计划]]。
