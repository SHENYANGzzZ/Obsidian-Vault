# 🎯 总策略（先记住这个）

👉 **只做3件事：**

1. 刷近5年真题（核心）
2. 背高频考点（少而精）
3. 练下午大题模板（拿分关键）

👉 目标：**45分稳过，不追高分**

------

# 🗓️ 30天详细计划（每天≈1.5~2小时）

------

## 🧠 第1阶段（Day 1–10）

👉 **目标：建立题感（不用全懂）**

### 每天：

- 上午题：1套选择题
- 下午题：1道大题（看解析为主）

### 📌 重点模块：

- 软件工程（最重要）
- 数据结构（固定题型）
- 数据库（下午必考）

### ❗做题原则：

👉 不会就看答案，不要卡

------

## 🔁 第2阶段（Day 11–20）

👉 **目标：掌握套路（关键阶段）**

### 每天：

- 上午题：1套（开始计时）
- 下午题：2道（开始总结模板）

### 📌 必做：

- 刷完**近5年真题**
- 开始整理错题本

------

## 🚀 第3阶段（Day 21–30）

👉 **目标：冲刺 + 稳定过线**

### 每天：

- 上午题：复刷错题 + 1套模拟
- 下午题：2道（只做高频题）

### 📌 重点：

👉 **背这些就够了：**

- 软件工程流程（瀑布 / 敏捷）
- UML图（类图、用例图）
- 设计模式（单例、工厂）
- SQL常见语句
- 数据结构（排序复杂度）

------

# 📊 下午大题（直接教你“拿分打法”）

👉 一共5道题，你只需要稳住3道！

------

## ✅ 必练3大题型

### 1️⃣ 数据库（必拿）

👉 SQL填空、主键外键
👉 **建议：每天练1题**

------

### 2️⃣ 软件工程

👉 甘特图 / 项目安排
👉 **记套路就能写**

------

### 3️⃣ 数据结构

👉 排序 / 树
👉 **规律性强**

------

# 💡 前端专属“捷径”

你可以直接利用已有经验：

- HTTP / 网络 → 上午题送分
- JS逻辑 → 数据结构更好上手
- 做过项目 → 软件工程更容易理解

👉 实际你比别人少学很多

------

# ⚠️ 关键避坑

❌ 不看厚教材
❌ 不做偏题怪题
❌ 不追求全懂

✅ 只做：
👉 **真题 + 高频考点**

------

# 🧨 最后7天（重点）

👉 每天只做这3件事：

1. 刷错题
2. 背高频
3. 练数据库 + 软件工程大题

👉 **不要再学新东西！**

------

# 🎯 考试当天策略

- 上午：不会就跳（别浪费时间）
- 下午：
  1. 先做数据库
  2. 再做软件工程
  3. 最后选简单题

👉 **目标：拿稳3道大题 = 通过**
