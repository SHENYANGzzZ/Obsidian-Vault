## 关键回答

### 接口文档内容提取

使用 textutil 读取 .doc 文件，获取到完整接口规范：

```
接口路径：/api/GDotComp（查询/新增/修改）
删除路径：/api/GDotComp/{id}
附件路径：/api/WRepair/uploadfilecomp（上传）
         /api/WRepair/downloadfilecomp（下载）
         /api/WRepair/deletefilecomp（删除）
```

### 字段映射调整

| 原设计字段    | 文档规范字段 | 说明             |
| ------------- | ------------ | ---------------- |
| nodE_NAME     | doT_NAME     | 节点名称         |
| nodE_TYPE     | doT_TYPE     | 类别             |
| companY_ID    | compid       | 公司ID           |
| -             | doT_CODE     | 节点代码（新增） |
| address       | -            | 已删除           |
| contacT_NAME  | -            | 已删除           |
| contacT_PHONE | -            | 已删除           |

### 附件管理修正

- **原设计**：4种附件类型（现场照片/规划图纸/合同文档/其他附件）
- **修正后**：3张照片（PIC1/PIC2/PIC3）

## 决策要点

- **决策1**：按文档规范修改所有接口路径
- **原因**：文档明确指定接口路径

- **决策2**：compid 设为必填
- **原因**：文档明确 compid 为必填，关联公司信息

- **决策3**：删除地址/联系人/联系电话字段
- **原因**：文档中无这些字段
