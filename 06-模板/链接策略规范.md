# Obsidian 链接策略规范

> 清晰、有意义、可持续的链接网络

## 🎯 核心原则

### 1. **有意义的连接**
- 每个链接都应该有明确的目的
- 避免为了链接而链接
- 链接应该帮助理解和导航

### 2. **适度的密度**
- 每篇笔记：3-7个有意义的链接
- 避免过度链接（>15个）
- 避免链接不足（<2个）

### 3. **双向链接优先**
- 优先建立双向链接
- 使用反向链接面板发现新连接
- 定期检查并补充反向链接

## 🔗 链接类型规范

### 1. **导航链接**（Navigation Links）
**用途**：在MOC和索引中提供快速导航

**格式**：
```markdown
- [[笔记名称]] - 简短说明
```

**示例**：
```markdown
- [[快速开始指南]] - 新手必读
- [[知识管理 MOC]] - 知识管理核心
- [[常见问题FAQ]] - 常见问题
```

**位置**：
- MOC文件的开头
- 主页和索引文件
- 项目概览文件

### 2. **概念引用链接**（Concept Reference Links）
**用途**：在笔记中引用相关概念和方法

**格式**：
```markdown
关于[[概念名称]]的理解...
```

**示例**：
```markdown
根据[[Zettelkasten方法]]的原则，每个笔记应该是原子化的...
```

**位置**：
- 笔记正文中的概念解释处
- 方法论引用处
- 理论依据处

### 3. **关联链接**（Association Links）
**用途**：在笔记末尾建立知识关联

**格式**：
```markdown
## 相关笔记

### 概念相关
- [[相关概念1]]
- [[相关概念2]]

### 方法相关
- [[相关方法1]]
- [[相关方法2]]

### 实践相关
- [[相关实践1]]
- [[相关实践2]]
```

**位置**：
- 笔记末尾
- 独立的"相关笔记"章节

### 4. **示例链接**（Example Links）
**用途**：在说明文档中作为示例

**格式**：
```markdown
<!-- 示例链接（不存在的笔记） -->
- [[示例笔记1]]
- [[示例笔记2]]
```

**特殊标记**：
- 使用HTML注释标记为示例
- 或使用特殊前缀：`[示例] [[笔记名称]]`

## 📝 链接使用场景

### 场景1：创建新笔记时

**立即添加的链接**：
1. **所属MOC**：`[[资源 MOC]]` 或 `[[项目 MOC]]`
2. **相关概念**：2-3个最相关的笔记
3. **同类笔记**：1-2个同类型的笔记

**示例**：
```markdown
# iTerm2配置与使用指南

> 终端利器，提升开发效率的快捷键和配置技巧

## 相关笔记

### 工具类
- [[Obsidian插件推荐]] - 其他效率工具
- [[Git常用命令]] - 版本控制工具

### 开发环境
- [[开发环境配置]] - 整体环境搭建
- [[VS Code配置]] - 编辑器配置
```

### 场景2：阅读笔记时

**发现新链接的时机**：
1. 遇到相关概念时
2. 发现可引用的内容时
3. 思考关联知识时

**操作**：
- 使用反向链接面板
- 定期回顾并补充链接

### 场景3：更新笔记时

**检查清单**：
- [ ] 链接是否仍然有效？
- [ ] 是否有新的相关笔记？
- [ ] 是否需要添加反向链接？
- [ ] 链接密度是否合适？

## 🚫 链接禁忌

### 1. **避免断链**
- 创建链接前确认目标笔记存在
- 使用Obsidian的链接建议功能
- 定期检查并修复断链

### 2. **避免过度链接**
- 不要每个句子都加链接
- 只链接真正相关的内容
- 避免重复链接同一笔记

### 3. **避免孤立笔记**
- 每个笔记至少有1-2个反向链接
- 使用MOC组织相关笔记
- 定期检查孤立笔记

### 4. **避免示例混淆**
- 明确标记示例链接
- 使用HTML注释或特殊前缀
- 不要让示例看起来像真实链接

## 🔧 链接维护流程

### 每周检查（5分钟）
1. **检查新笔记的链接**：
   - 是否添加了相关链接？
   - 链接密度是否合适？

2. **检查反向链接**：
   - 是否有新的反向链接？
   - 是否需要补充？

### 每月检查（15分钟）
1. **检查断链**：
   - 使用Obsidian的断链检查功能
   - 修复或删除断链

2. **检查孤立笔记**：
   - 查找没有反向链接的笔记
   - 为孤立笔记添加链接

3. **优化链接密度**：
   - 检查链接过多的笔记
   - 检查链接过少的笔记

### 每季度检查（30分钟）
1. **评估链接质量**：
   - 链接是否有意义？
   - 是否帮助理解和导航？

2. **优化MOC结构**：
   - MOC是否覆盖所有内容？
   - MOC之间的链接是否清晰？

3. **更新链接规范**：
   - 是否需要调整规范？
   - 是否有新的最佳实践？

## 📊 链接质量指标

### 健康度指标
- **断链率**：< 5%（目标：0%）
- **孤立笔记率**：< 5%（目标：< 3%）
- **链接覆盖率**：> 90%（目标：> 95%）

### 密度指标
- **平均链接数**：3-7个/笔记
- **链接过多**：> 15个/笔记
- **链接不足**：< 2个/笔记

### 质量指标
- **有意义链接比例**：> 90%
- **双向链接比例**：> 70%
- **MOC覆盖率**：100%

## 🛠️ 工具和技巧

### Obsidian内置功能
1. **反向链接面板**：发现未建立的链接
2. **出链面板**：检查笔记中的所有链接
3. **未链接提及**：发现可以建立链接的地方
4. **断链检查**：查找并修复断链

### 插件推荐
1. **Dataview**：动态查询和统计
2. **Graph Analysis**：分析链接网络
3. **Broken Links**：自动检查断链

### 快捷键
- `Cmd + K`：插入链接
- `Cmd + Shift + K`：插入外部链接
- `Cmd + Option + L`：打开反向链接面板

## 📋 链接检查清单

### 创建笔记时
- [ ] 添加了所属MOC链接
- [ ] 添加了2-3个相关概念链接
- [ ] 添加了1-2个同类笔记链接
- [ ] 链接密度适中（3-7个）

### 更新笔记时
- [ ] 检查了现有链接的有效性
- [ ] 补充了新发现的相关链接
- [ ] 检查了反向链接
- [ ] 优化了链接密度

### 定期维护时
- [ ] 修复了所有断链
- [ ] 处理了孤立笔记
- [ ] 优化了链接质量
- [ ] 更新了MOC链接

## 🎯 执行计划

### 第一阶段：修复断链（本周）
1. 创建缺失的笔记文件
2. 修复命名不一致的链接
3. 删除或标记示例链接

### 第二阶段：优化链接（本月）
1. 为孤立笔记添加链接
2. 优化链接密度
3. 完善MOC链接

### 第三阶段：建立习惯（持续）
1. 创建笔记时立即添加链接
2. 定期检查和维护
3. 持续优化链接质量
