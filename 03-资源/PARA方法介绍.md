# PARA 方法介绍

## 什么是 PARA？

PARA 是由 Tiago Forte 提出的信息组织方法，适用于个人知识管理和数字笔记系统。

- **P**rojects（项目）- 有明确截止日期的工作
- **A**reas（领域）- 需要持续关注的责任范围
- **R**esources（资源）- 有价值的参考资料
- **A**rchive（归档）- 已完成或不再活跃的内容

---

## 四要素详解

### 1. Projects（项目）

**定义**：有明确目标和截止日期的一次性工作

**特征**：
- 有明确的完成标准
- 有时间限制
- 完成后不再持续

**示例**：
- 完成产品发布会
- 写一本书
- 开发一个 App
- 准备考试

**存放位置**：`01-项目/`

---

### 2. Areas（领域）

**定义**：需要持续投入和关注的责任范围

**特征**：
- 没有明确的截止日期
- 需要长期维护
- 有标准和习惯要求

**示例**：
- 职业发展
- 健康管理
- 财务管理
- 家人关系

**存放位置**：`02-领域/`

---

### 3. Resources（资源）

**定义**：有价值但不是立即需要的参考资料

**特征**：
- 暂时不活跃
- 可能将来有用
- 不属于某个项目或领域

**示例**：
- 感兴趣的话题
- 工具教程
- 收藏的文章
- 学习资料

**存放位置**：`03-资源/`

---

### 4. Archive（归档）

**定义**：已完成或不再活跃的内容

**特征**：
- 项目已完成
- 领域不再相关
- 资源已过时

**示例**：
- 已完成的项目
- 旧的参考资料
- 已放弃的想法

**存放位置**：`04-归档/`

---

## PARA vs 传统文件夹

| 传统文件夹 | PARA |
|-----------|------|
| 按主题分类 | 按行动分类 |
| 容易堆积 | 定期流动 |
| 难找相关内容 | 围绕工作组织 |
| 静态 | 动态 |

---

## PARA 的流动

```
Inbox（收集箱）
    ↓ [处理]
项目 / 领域 / 资源
    ↓ [完成后]
Archive（归档）
```

**原则**：信息应该在 PARA 各部分之间流动，而不是静态存放。

---

## 在 Obsidian 中的实现

### 目录结构

```
00-收集箱/     ← 所有想法的入口
01-项目/       ← 当前项目
02-领域/       ← 关注的领域
03-资源/       ← 参考资料
04-归档/       ← 已完成内容
```

### 使用原则

1. **一个笔记，一个归属**：每条笔记应该主要属于一个项目或领域
2. **链接代替移动**：使用双向链接代替移动文件
3. **定期整理**：每周检查 PARA 各部分
4. **归档而非删除**：旧内容移到归档而不是删除

---

## 实践建议

### 每天
- 处理 Inbox
- 更新项目进展

### 每周
- 回顾项目和领域
- 整理资源
- 归档已完成内容

### 每月
- 全面审视 PARA 结构
- 调整分类
