# Git工作流

## 结论

日常协作优先使用“小步提交、短分支、尽快合并”的工作流，不要让分支长期漂移。

## 推荐做法

- 新需求或修复从独立分支开始。
- 每次提交只解决一个清晰问题。
- 提交前先自测，减少把明显问题带进主线。
- 分支存活时间尽量按天算，不按周算。

## 常见步骤

1. 从最新主线切分支。
2. 本地开发并分批提交。
3. 同步主线变更，处理冲突。
4. 发起合并并完成复核。
5. 合并后删除分支。
