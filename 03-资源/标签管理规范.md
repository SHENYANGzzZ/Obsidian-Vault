# 标签管理规范

## 标签 vs 链接 vs MOC

| 工具 | 用途 | 示例 |
|------|------|------|
| **链接** | 笔记间的精确关系 | `[[Zettelkasten方法]]` |
| **标签** | 粗略分类、多维度标记 | `#方法论 #zettelkasten` |
| **MOC** | 组织和导航主题笔记 | [[知识管理 MOC]] |

> **原则**：优先使用链接，标签用于辅助分类

---

## 标签层级结构

### 一级标签（类别）

| 标签 | 含义 | 使用场景 |
|------|------|---------|
| `#永久笔记` | 经过思考的沉淀知识 | 知识库的核心内容 |
| `#闪念笔记` | 临时想法、灵感 | Inbox 中的原始想法 |
| `#文献笔记` | 来自外部资源 | 文章、书籍、课程的摘录 |
| `#项目` | 项目相关 | 项目文档、任务 |
| `#领域` | 长期关注的领域 | 健康、财务、技能 |
| `#资源` | 参考资料 | 工具、教程、链接 |

### 二级标签（子类别）

格式：`#类别/子类别`

示例：
- `#项目/feishu-shadow` - 特定项目
- `#领域/知识管理` - 特定领域
- `#资源/插件` - 特定类型的资源

### 状态标签

| 标签 | 含义 |
|------|------|
| `#status/进行中` | 正在进行 |
| `#status/待办` | 还没开始 |
| `#status/已完成` | 已经完成 |
| `#status/搁置` | 暂停或放弃 |

### 元标签

| 标签 | 含义 |
|------|------|
| `#moc` | MOC 页面 |
| `#模板` | 模板文件 |
| `#日记` | 每日日记 |
| `#对话` | AI 对话记录 |

## 标签命名规则

### ✅ 好的标签命名

```
#永久笔记
#闪念笔记
#项目/feishu-shadow
#领域/知识管理
#status/进行中
#2026-03
```

### ❌ 不好的标签命名

```
#笔记          # 太模糊
#重要          # 含义不清
#todo          # 混合语言
#AAA           # 无意义
```

## 标签使用场景

### 什么时候使用标签？

1. **多维度分类**：一条笔记属于多个类别
2. **快速筛选**：想快速找到某类所有笔记
3. **时间标记**：标记创建或修改时间
4. **状态跟踪**：跟踪任务或项目状态

### 什么时候不用标签？

1. **已有明确链接**：笔记间已通过链接关联
2. **单一主题**：笔记只属于一个主题
3. **可以用 MOC**：导航目的用 MOC 更合适

## 标签数量控制

### 建议

- 每个笔记 1-3 个主要标签
- 避免超过 5 个标签
- 定期清理无用标签

### 标签审计

每月底检查：
1. 列出所有标签：`Dataview TABLEDistinct(tags) FROM ""`
2. 合并相似标签
3. 删除从未使用的标签

## 常见问题

**Q: 一个笔记应该打几个标签？**
A: 建议 1-3 个。标签太多说明分类不清晰。

**Q: 标签和文件夹冲突吗？**
A: 不冲突。文件夹用于物理组织，标签用于逻辑分类。一条笔记可以在任何文件夹，同时拥有任何标签。

**Q: 如何避免标签混乱？**
A: 制定规范后坚持执行，定期审计和合并。
