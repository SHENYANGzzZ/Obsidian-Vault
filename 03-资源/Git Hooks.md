# Git Hooks

## 结论

Git Hooks 适合拦截低成本、可自动判断的问题，例如格式化、基础校验和提交信息检查。

## 常见场景

- `pre-commit`：格式化、lint、快速测试。
- `commit-msg`：校验提交信息格式。
- `pre-push`：补一层稍重但仍可接受的检查。

## 使用建议

- Hook 要快，慢了就会被绕过。
- 只做稳定、自动化程度高的检查。
- 团队统一时，优先用脚本和配置文件管理，不靠个人手工复制。
