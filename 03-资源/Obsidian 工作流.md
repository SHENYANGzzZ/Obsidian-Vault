# Obsidian 工作流

## 结论

这个仓库更适合“先记录，再归档，再沉淀”的轻流程，而不是一开始就做复杂分类。

## 推荐顺序

1. 新内容先进入 `05-每日/` 或 `00-收集箱/`。
2. 每周整理到对应项目的 `worklog/`。
3. 能长期复用的内容再沉淀为 `03-资源/`、`02-领域/` 或 MOC。

## 判断标准

- 只是时间性记录：留在日志。
- 已形成稳定方法：转资源笔记。
- 是项目长期决策：进入项目 README 或 plans。
