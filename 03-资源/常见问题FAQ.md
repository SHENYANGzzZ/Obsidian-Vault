# 常见问题 FAQ

## 基础问题

### Q1: 这个知识库是用什么工具管理的？
**A**: 使用 [Obsidian](https://obsidian.md/) 管理，它是一个强大的本地知识库工具，支持 Markdown 和双向链接。

---

### Q2: PARA 是什么？
**A**: PARA 是组织信息的四种类型：
- **P**rojects（项目）- 有明确截止日期的工作
- **A**reas（领域）- 需要持续关注的事
- **R**esources（资源）- 有价值的参考材料
- **A**rchive（归档）- 已完成的内容

详见：[[知识管理的本质是建立连接]]

---

### Q3: Zettelkasten 是什么？
**A**: Zettelkasten（卡片盒笔记法）是一种知识管理方法，核心是：
- 原子化：一条笔记一个想法
- 链接：笔记之间建立连接
- 自下而上：从具体笔记涌现主题

详见：[[Zettelkasten方法]]

---

## Inbox 相关

### Q4: Inbox 是什么？为什么重要？
**A**: Inbox（收集箱）是所有想法的入口。任何时刻产生的想法都先存入这里，避免打断当前工作，稍后统一处理。

详见：[[收集箱处理工作流]]

---

### Q5: 闪念笔记和永久笔记有什么区别？
**A**:
| 类型 | 特征 | 处理 |
|------|------|------|
| 闪念笔记 | 原始想法 | 临时存放，快速捕获 |
| 永久笔记 | 思考后的知识 | 沉淀、链接、永久保存 |

---

### Q6: Inbox 堆积了太多内容怎么办？
**A**: 
1. 设定每天固定处理时间
2. 批量处理，每条不超过 2 分钟
3. 快速决策：转化、删除、或保留
4. 坚持每天清空，不堆积

---

## 链接和标签

### Q7: 双向链接是什么？
**A**: 用 `[[笔记标题]]` 语法创建的链接。不仅能从当前笔记跳转到目标笔记，还能在目标笔记的反向链接面板中看到当前笔记。

详见：[[笔记链接的艺术]]

---

### Q8: MOC 是什么？
**A**: MOC（Map of Content）是内容地图，用于组织和导航相关主题的笔记。它像一本书的目录，但更加灵活和非线性。

详见：[[MOC的作用]]

---

### Q9: 标签和链接有什么区别？
**A**:
| 特性 | 链接 | 标签 |
|------|------|------|
| 关系 | 明确指出相关性 | 粗略分类 |
| 导航 | 精确跳转 | 模糊筛选 |
| 组织 | 形成网络 | 形成类别 |

建议：优先使用链接，标签用于辅助分类。

详见：[[标签管理规范]]

---

### Q10: 什么时候用标签？什么时候用链接？
**A**:
- **链接**：笔记之间有明确关联时
- **标签**：粗略分类、多维度标记时
- **MOC**：组织多个相关笔记时

---

## 每日笔记

### Q11: 每日笔记有什么用？
**A**: 
- 记录当天的想法、收获、行动
- 作为时间戳，方便回顾
- 连接长期笔记和短期记录

详见：[[每日笔记模板]]

---

### Q12: 如何养成每日笔记的习惯？
**A**:
1. 设置固定的记录时间（晨间或晚间）
2. 使用模板降低启动阻力
3. 从简单开始，不要追求完美
4. 定期回顾，增强动力

---

## 项目管理

### Q13: 什么算"项目"？
**A**: 项目是有明确目标和截止日期的工作。特征：
- 有明确的产出
- 有明确的完成时间
- 可以分解为具体任务

详见：[[项目模板]]

---

### Q14: 项目完成后怎么办？
**A**:
1. 更新项目状态为"已完成"
2. 记录最终成果和教训
3. 移动到 `04-归档`
4. 更新 [[项目 MOC]]

---

## 同步和备份

### Q15: 如何同步笔记到其他设备？
**A**: 使用 Git 同步：
1. 在 GitHub/Gitee 创建仓库
2. 本地初始化 Git
3. 使用 Obsidian Git 插件自动同步

详见：[[使用Git同步Obsidian]]

---

### Q16: 重要内容如何备份？
**A**:
1. 使用 Git 同步到云端仓库
2. 设置 Obsidian Git 插件的自动提交
3. 定期检查同步状态

---

## 其他问题

### Q17: 笔记太多，找不到想要的内容怎么办？
**A**:
1. 使用 `Ctrl/Cmd + O` 快速搜索
2. 使用 Dataview 查询
3. 通过 MOC 导航
4. 定期整理，避免堆积

---

### Q18: 如何保持知识库长期有效？
**A**:
1. 定期回顾（每周、每月）
2. 持续处理 Inbox
3. 及时建立链接
4. 更新和删除过时内容
5. 记录自己的思考，不要只收藏
