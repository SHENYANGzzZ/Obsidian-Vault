# 快速开始指南

> 这份指南只保留当前仓库还在使用的流程。

## 第一步：了解核心理念

建议先阅读：

1. [[知识管理的本质是建立连接]]
2. [[原子笔记的价值]]
3. [[链接比文件夹更重要]]

## 第二步：熟悉目录结构

```text
├── 00-收集箱/     ← 临时收集
├── 01-项目/       ← 项目主目录
├── 02-领域/       ← 长期关注领域
├── 03-资源/       ← 参考资料
├── 04-归档/       ← 已归档内容
├── 05-每日/       ← 每日原始记录入口
├── 06-模板/       ← 当前使用中的模板与规范
└── MOCs/          ← 索引页
```

详细说明见根 README。

## 第三步：日常使用

### 3.1 记录每日工作

1. 在 `05-每日/` 新建当天笔记
2. 使用 [[每日日记模板]]
3. 项目工作按 `## [[项目名]]` 分块记录
4. 其他生活、学习、闪念内容写在“其他记录”里

### 3.2 整理项目记录

1. 每周回顾本周每日笔记
2. 按项目把内容归档到对应项目目录的 `worklog/`
3. 长周期计划或专题方案写进 `plans/`
4. 对公司输出的内容写进 `weekly-reports/`

### 3.3 建立链接

- 目录能表达清楚时，优先靠目录，不补多余链接
- 只有确实缩短跳转路径时，才使用 wiki 链接
- 具体规则见 [[链接策略规范]]

## 第四步：常见场景

| 场景 | 操作 |
| --- | --- |
| 日常开发 | 先写每日，再周整理到项目 `worklog/` |
| 项目启动 | 在 `01-项目/` 下创建项目目录和 README |
| 阶段规划 | 写到项目 `plans/` |
| 对外周报 | 写到项目 `weekly-reports/` |
| 项目结束 | 移动到 `04-归档/` |
