## 遇到的问题

### 问题1：Trae 安全限制无法直接写入 Obsidian 库

- **原因**：Trae 的编辑操作限制在工作目录内
- **解决方案**：文件先保存到项目目录，提供复制命令让用户手动复制
- **相关代码**：
  ```bash
  cp -r obsidian-sync/* /Users/shenyang/Documents/Obsidian-Vault/01-项目/feishu-shadow/
  ```

### 问题2：Obsidian 知识库存在断链

- **原因**：笔记引用了不存在的笔记
- **解决方案**：创建缺失的笔记文件（如何写好笔记标题、笔记链接的艺术等）

## 技术决策

### 决策1：使用 Python 实现同步脚本

- **原因**：Python 生态丰富，易于处理 Markdown 和文件操作
- **影响**：需要 Python 环境，但大多数开发者已有
- **替代方案**：Shell 脚本（可移植性差）、Node.js（需要额外依赖）

### 决策2：按项目组织文件结构

- **原因**：符合开发工作流，便于项目管理
- **影响**：需要创建项目目录结构
- **替代方案**：按日期组织（不便于项目追踪）

### 决策3：混合触发模式

- **原因**：平衡自动化和可控性
- **影响**：重要内容自动同步，其他需要手动触发
- **实现**：通过关键词检测和用户确认
